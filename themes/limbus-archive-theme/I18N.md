# i18n Support - Limbus Archive Theme

The Limbus Archive theme now includes full internationalization (i18n) support.

## Supported Languages

Currently supported languages:

- **English** (`en`) - Default
- **简体中文** (`zh-cn`) - Simplified Chinese
- **日本語** (`ja`) - Japanese

## Quick Setup

### Single Language Site (Default)

If you only need one language, no special configuration is needed. The theme will use English by default.

### Multilingual Site

To enable multiple languages, use the multilingual configuration:

#### 1. Copy the Multilingual Config

```bash
cp themes/limbus-archive-theme/exampleSite/config-multilingual.toml config.toml
```

#### 2. Organize Your Content

Create language-specific content directories:

```
content/
├── posts/              # English content (default language)
│   └── hello.md
├── about.md
│
├── zh-cn/              # Chinese content
│   ├── posts/
│   │   └── hello.md
│   └── about.md
│
└── ja/                 # Japanese content
    ├── posts/
    │   └── hello.md
    └── about.md
```

#### 3. Create Translated Content

**English post** (`content/posts/hello.md`):
```yaml
---
title: "Hello World"
date: 2025-03-18
---

This is an English post.
```

**Chinese post** (`content/zh-cn/posts/hello.md`):
```yaml
---
title: "你好世界"
date: 2025-03-18
---

这是一篇中文文章。
```

**Japanese post** (`content/ja/posts/hello.md`):
```yaml
---
title: "こんにちは世界"
date: 2025-03-18
---

これは日本語の記事です。
```

## Configuration Details

### Language Configuration

```toml
[languages]
  [languages.en]
    languageName = "English"
    weight = 1              # Display order
    title = "Limbus Archive"
    
    [languages.en.params]
      description = "Your description"
      footer = "© 2025 Your Name"
      mainSections = ["posts"]
```

### Menu Configuration

Each language can have its own menu:

```toml
[[languages.en.menu.main]]
  name = "Home"
  url = "/"
  weight = 1

[[languages.zh-cn.menu.main]]
  name = "首页"
  url = "/zh-cn/"
  weight = 1
```

## URL Structure

### Default Language in Root

```toml
defaultContentLanguage = "en"
defaultContentLanguageInSubdir = false
```

URLs:
- English: `https://yoursite.com/posts/hello/`
- Chinese: `https://yoursite.com/zh-cn/posts/hello/`
- Japanese: `https://yoursite.com/ja/posts/hello/`

### All Languages in Subdirectories

```toml
defaultContentLanguageInSubdir = true
```

URLs:
- English: `https://yoursite.com/en/posts/hello/`
- Chinese: `https://yoursite.com/zh-cn/posts/hello/`
- Japanese: `https://yoursite.com/ja/posts/hello/`

## Language Switcher

The theme automatically displays a language switcher in the left sidebar when multiple languages are configured.

The current language appears as a pressed button (debossed), while other languages are clickable (embossed).

## Adding New Languages

### 1. Create Translation File

Create `i18n/[language-code].toml` (e.g., `i18n/fr.toml` for French):

```toml
[home]
other = "Accueil"

[posts]
other = "Articles"

[read_more]
other = "Lire la suite →"

# ... other translations
```

### 2. Add Language to Config

```toml
[languages.fr]
  languageName = "Français"
  weight = 4
  title = "Archive Limbus"
  
  [languages.fr.params]
    description = "Une archive inspirée des terminaux CRT"
    footer = "© 2025 Archive Limbus"
    mainSections = ["posts"]
  
  [[languages.fr.menu.main]]
    name = "Accueil"
    url = "/fr/"
    weight = 1
```

### 3. Create Content

```
content/
└── fr/
    ├── posts/
    │   └── bonjour.md
    └── about.md
```

## Translation Keys Reference

All available translation keys are in:
- `i18n/en.toml` - English (reference)
- `i18n/zh-cn.toml` - Simplified Chinese
- `i18n/ja.toml` - Japanese

### Common Keys

| Key | English | 中文 | 日本語 |
|-----|---------|------|--------|
| `home` | Home | 首页 | ホーム |
| `posts` | Posts | 文章 | 記事 |
| `read_more` | Read More → | 阅读更多 → | 続きを読む → |
| `archive` | Archive | 归档 | アーカイブ |
| `tags` | Tags | 标签 | タグ |
| `about` | About | 关于 | について |

## Disable Language Switcher

If you want to hide the language switcher, edit `layouts/_default/baseof.html` and remove:

```html
{{ partial "language-switcher.html" . }}
```

## Per-Page Language

You can specify a page's language in front matter:

```yaml
---
title: "Title"
date: 2025-03-18
language: en
---
```

## RSS Feeds

Each language gets its own RSS feed:

- English: `/index.xml`
- Chinese: `/zh-cn/index.xml`
- Japanese: `/ja/index.xml`

## Deployment Considerations

### Netlify

Your `netlify.toml` works automatically with multilingual sites:

```toml
[build]
  publish = "public"
  command = "hugo --minify"

[build.environment]
  HUGO_VERSION = "0.120.0"
```

### GitHub Pages

No special configuration needed. The GitHub Actions workflow handles multilingual builds automatically.

## Common Issues

### Language Switcher Not Showing

- Make sure you have more than one language configured in `config.toml`
- Check that `[languages]` section exists

### Translations Not Working

- Verify the i18n file name matches your language code
- Check that the translation key exists in your i18n file
- Ensure `defaultContentLanguage` is set correctly

### Menu Links Wrong

- Update menu URLs for each language
- Use absolute paths with language prefix: `/zh-cn/posts/`

## Best Practices

1. **Consistent structure**: Keep the same content structure across all languages
2. **Fallback content**: Provide default language content even if translations are incomplete
3. **Date formats**: Hugo automatically formats dates based on language
4. **URLs**: Use descriptive URLs in each language
5. **SEO**: Set `hreflang` tags (theme handles this automatically)
